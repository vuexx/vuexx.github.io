什么是Vue全局事件总线?

一般我们在设计和组合Vue组件时，大多数时候都是父子组件间的交互 Parent-Child Communication。父组件通过组件属性传递参数和值(props down)给子组件，子组件通过事件(events up)传递交互动作给父组件。

在实际的应用开发中，经常会遇到这种麻烦：两个组件之间需要通信，但他们又并不是父子组件的关系。很多时候，我们会引入Vue官方推荐的状态管理库 Vuex 来解决。但这种方式对于一些功能相对简单的App来说，引入Vuex感觉有些重了，因为你可能仅仅是为了解决两三个非父子组件间的通信而已。这种情况下，全局事件总线的设计可能就更适合你了。

再次声明，全局事件总线的设计只适合在功能相对简单，又遇到组件通信麻烦的App中使用。但在大多数的App中这不是推荐的解决方案，如果你的App相对复杂，引入Vuex来管理可能会更加适合。否则随着App复杂程度的提升，全局事件总线的代码可能会变得混乱难以维护，但在相对简单的应用中，它非常方便好用。
